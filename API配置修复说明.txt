========================================
问题原因：
========================================

前端代码验证太严格：
- 即使服务器端已经配置了Kimi的API Key
- 前端仍然要求必须提供API Key
- 导致显示"当前AI未配置API Key"错误

========================================
修复内容：
========================================

移除了前端的严格API Key验证：

**修改前：**
```javascript
if (!providerData || !providerData.config || !providerData.config.apiKey || 
    providerData.config.apiKey === 'your-api-key') {
    addMessage('❌ 错误：当前AI未配置API Key...', 'assistant', true);
    return;
}
```

**修改后：**
```javascript
if (!providerData) {
    addMessage('❌ 错误：无法识别的模型类型', 'assistant', true);
    return;
}
// 允许发送空配置，使用服务器端的默认配置
```
